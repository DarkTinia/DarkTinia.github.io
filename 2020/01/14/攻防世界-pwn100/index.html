64位程序，开了堆栈不可执行

放到ida静态分析

c
c
c

程序逻辑就是向v1里读200个字符，然而v1到栈底的距离只有0x40，也就是说存在栈溢出
看一下函数窗口和字符串窗口，发现既没有”/bin/sh”，也没有system。但程序用到了put和read，可以利用put构造leak实现任意地址读，利用read写入”/bin/sh”
因为64位程序前六个参数分别保存在 rdi rsi rdx rcx r8 r9，所以我们需要想办法改变寄存器的值。利用ROPgadget搜索

Code

在init里找到两段gadget

Code

仔细观察pop rdi的地址，是不是有点奇怪
源程序中400762处代码为pop r15，紧接着就是400764处的retn，而pop rdi的地址是400763
按D将pop r15转换成数据，再将第二行按C转换成代码。pop rdi出现了

Code

这再一次验证了代码本身就是一串数据，只是和其他数据解读方式不一样

现在有了两段gadget和pop rdi，从文件中能找到put和read，也就是说具备了写leak的条件。然后利用leak泄露地址计算偏找到system，再调用system获取shell
只差”/bin/sh”了。在ida按ctrl+S发现601050这段可以写入。
万事俱备，整理下思路。首先构造一个leak，利用DynELF获取system函数地址。然后调用read函数写入”/bin/sh”，最后调用system函数。
