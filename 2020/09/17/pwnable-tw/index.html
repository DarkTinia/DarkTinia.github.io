但没有canary。没办法直接溢出。这里利用了一个特性。

scanf(“%u”,&a);当输入一个‘f’时scanf并不会读取该字符，并且该字符会滞留在stdin的缓冲区中。下次scanf时，由于在缓冲区读到的仍是‘f’，scanf依旧不会读取该字符，如此反复。但当我们输入’+’时，由于不是数字，所以scanf不会读取，当我们下一次读入一个数字时，缓冲区会连带之前的正号一起读入到a中。那么我们就可以利用+来绕过canary。

applestore

一道32位伪堆题。题目自己实现了unlink。

c

add时会malloc一块0x10大小的堆，delete时并未free掉，而是将被delete的块解链。

c

checkout时若金额恰好为7174则会在链中加入一个栈中的chunk。而该chunk恰好可被my_read覆写到。

题目中chunk的结构为 name | num | fd | bk ，那么将栈中的chunk的name改成某个got即可泄露libc。

然后就不知道该怎么做了。

查了其他大佬的做法，发现是利用libc中的environ来泄露stack，进而能计算出delete函数的ebp，

然后将栈中的chunk修改为 name | num | atoi_got+0x22 | ebp - 0x8

这样当delete函数执行完解链操作后，ebp处存放的值会被修改为atoi_got + 0x22，delete函数执行结束后，ebp就会指向atoi+0x22。下次在handle函数中调用my_read就可以恰好修改到atoi_got。此时输入p32(system)+’;/bin/sh\x00’即可getshell。
